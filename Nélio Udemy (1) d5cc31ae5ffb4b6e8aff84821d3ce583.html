O que é um compilador ?

Compilador: É um programa que realiza a conversão de linguagem de alto nível para baixo nível.

Linguagem de alto nível: A Linguagem que entendemos e escrevemos nosso código fonte.

Linguagem de baixo nível: A linguagem que a máquina entende. Possui pouca abstração, sendo difícil de entender.

Compilador e Transpilador

Compilador: é um programa que realiza a conversão de linguagem de alto nível para baixo nível. Exemplo : C#, Java.

Transpilador: É a conversão de uma linguagem ou implementação para outra. A sua saída permanece em linguagem de alto nível. Exemplo: TypeScript para JavaScript.

Linguagem compilada: São linguagens o código fonte é traduzido para o código de máquina. Exemplo: C#, Java

Linguagem interpretada: São linguagens que fazem a leitura e interpretação diretamente do código fonte. Exemplo: JavaScript, PHP.

O que é uma programação orientada a objetos

É um modelo de programação baseada no conceito de objetos que podem conter dados na forma de atributos e métodos, nesse processo de programação são criadas coleções de objetos com estruturas e comportamentos próprios, esses objetos interajam entre si, e executam as ações solicitadas, o objetivo é basicamente a unificação de dados e processos e o agrupamento e reutilização de códigos. A programação orientada a objetos é fundamentada em 4 estruturas mestres e são elas: Classes, Objetos, Atributos e Métodos, e também temos os principais pilares da POO, sendo eles Encapsulamento, Herança e Polimorfismo.

O que é uma Classe ?

Classe é um conjunto de objetos que se definem pelo fato desses objetos em questão terem mais de uma característica em comum

O que é um Objeto ?

O objeto pode ser qualquer coisa instanciada a partir de uma classe, ele é um elemento pertencente de uma classe.

O que são Atributos ?

Atributos é um conjunto de características especificas de um determinado objeto.

O que é Método ?

É a característica que permite alterar a funcionada de cada atributo, é por meio dos métodos que se é possível modificar os atributos de um determinado objeto.

Conversão

Depois que uma variável é declarada, ela não pode ser declarada novamente ou atribuída a um valor de outro tipo, a menos que esse tipo possa ser convertido implicitamente no tipo da variável. Por exemplo, a string não pode ser convertida implicitamente em int. Portanto, depois de declarar

“x” como um int, não é possível atribuir a cadeia de caracteres. E para isso usamos a Conversão.

Para fazermos a Conversão, usamos o seguinte comando :

De string para int : int.Parse(variável); , esse comando converte a representação de cadeia de caracteres de um número no inteiro com sinal de 32 bits equivalente.

int n1 = int.Parse(Console.ReadLine());

De string para char : char.Parse(variável);

char ch = char.Parse(Console.ReadLine());

De string para float : float.Parse(variável);

nota1 = float.Parse(txtNota1);

De string para double : double.Parse(variável);

double n2 = double.Parse(Console.ReadLine());

De int para string : Convert.ToString(variável); ou variável.ToString( ); , esse comando converte o valor inteiro com sinal de 32 bits especificado em sua representação de cadeia de caracteres equivalente.

lblFrase.Text = Convert.ToString(frase.Length);

//ou 

lblFrase.Text = frase.Length.ToString( );

Funções representam um processo que possui um significado.

Suas principais vantagens: modularização, delegação e reaproveitamento.

  • Modularização é dividir o seu programa em partes menores.
  • Delegação você vai delegar aquele processamento para um outro lugar, deixando assim o seu programa mais simples.
  • Reaproveitamento quando você tem uma função que já está pronta, você só chama está função ao invés de repetir toda aquela função de novo.

Dados de entrada e saída :

  • As funções de entrada podem receber dados de entradas que são chamados de parâmetros ou argumentos.
  • Funções de saída que podem ou não retornar uma saída, está saída é o resultado da função.

Em orientação de objetos, funções em classes recebem o nome de “métodos”.

internal class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Digite 3 números : ");
        int n1 = int.Parse(Console.ReadLine());
        int n2 = int.Parse(Console.ReadLine());

        Console.ReadLine();

        }

        }

Método

Um Método sempre terá no final os parênteses ( ).

Sempre que criamos um Método por exemplo do tipo string ou double, o próprio C# espera um retorno, então sempre após digitarmos o código dentro dessa string temos que declarar um return ao final.

Os membros estáticos, são membro que fazem sentido independentemente de objetos. Não precisam de objeto para serem chamados. São chamados a partir do próprio nome da classe.

Tipo Referência e Tipo Valor

Classes são tipo referência

Variáveis cujo tipo são classes não devem ser entendidas como caixas, mas sim “tentáculos”(ponteiros) para caixas.

Valor “null”

Tipos de referência aceitam o valor “null”, que indica que a variável aponta para ninguém.

Structs são tipos Valor

A linguagem C# possui também tipos valor, que são os “structs”. Structs são CAIXAS e não ponteiros.

Em Struct ele copia o mesmo valor de X, isso não quer dizer que ele aponta para X mas sim que ele copia

Valores Padrão

Quando alocamos (new) qualquer tipo estruturado (classe, strut, array), são atribuídos valores padrão aos seus elementos.

  • Números: 0
  • Bool : False
  • Char : caractere código 0, não corresponde ao dígito 0.
  • Objeto : null

Lembrando: uma variável apenas declarada, mas não instanciada, inicia em estado “não atribuído”, e o próprio compilador não permite que ela seja acessada.

Garbage collector

É um processo que automatiza o gerenciamento de memória de um programa em execução. O garbage collector monitora os objetos alocados dinamicamente pelo programa (no heap), desalocando aqueles que não estão mais sendo utilizados.

Desalocação por escopo

Pensando em um programa que tem x = 10, e nesse mesmo programa contém um if que por determinado parâmetro y = 20, após a saída desse escopo, esse y = 20, passa a ser desalocado, e isso é a desalocação por escopo.

Nullable

É um recurso de C# para que dados de tipo valor (structs) possam receber o valor null.

Uso Comum :

  • Campos de banco de dados que podem valer nulo (data de nascimento, algum valor numérico, etc.)
  • Dados e Parâmetros opcionais.

Nullable<double> x = null; ou double? x = null;

O Nullable possui 3 operações básicas (métodos), sendo elas:

  • GetValueOrDefault, esse método diz o seguinte pegue o valor de X ou pegue o valor padrão, no caso acima o valor não existe, logo o valor padrão de um double é 0
  • HasValue
  • Value (lança uma exceção se não houver valor)

Um Nullable não pode ser atribuído para um struct comum.

class Program
{
		static void Main(string[] args) 
		{
			double? x = null;
			double? y = 10.0;

			Console.WriteLine(x.GetValueOrDefault()); //esse método diz o seguinte pegue o valor de X ou pegue o valor padrão, no caso acima o valor não existe, logo o valor padrão de um double é 0
			Console.WriteLine(y.GetValueOrDefault()); //no caso de y, ele retorna o 10.0

			Console.WriteLine(x.HasValue); //retornou falso, porque o x não tem valor
			Console.WriteLine(y.HasValue); //retornou true, poque o y tem valor de 10.0

			if (x.HasValue)
				Console.WriteLine(x.Value); //retorno se fosse isolado, daria um crash, pois, x, não tem valor
			else
				Console.WriteLine("X is null");

			if (y.HasValue)
				Console.WriteLine(y.Value); 
			else
				 Console.WriteLine("Y is null");
		}
}

Tratamento de Exceções

Exceção é qualquer condição de erro ou comportamento inesperado encontrado por um programa em execução.

No .NET, uma exceção é um objeto herdado da classe System.Exception

Quando lançada, uma exceção é propagada na pilha de chamadas de métodos em execução, até que seja capturada(tratada) ou o programa seja encerrado.

Tipo Exception, é um tipo da linguagem c#, ele possui subtipos, a partir do Exception, temos duas classes principais, sendo elas, SystemException e ApplicationException.

A existência dessas duas classes é uma forma de permitir diferenciar exceções de sistema e exceções da sua aplicação.

  • SystemException, são as exceções geradas pelo .NET, pelas bibliotecas da .NET.
  • ApplicationException, você pode criar suas exceções personalizadas.

Por que existem as Exceções?

O modelo de tratamento de exceções permite que erros sejam tratados de forma consistente e flexível, usando boas práticas.

Vantagens:

  • Delega a lógica do erro para a classe/método responsável por conhecer as regras que podem ocasionar o erro
  • Trata de forma organizada(inclusive hierárquica) exceções de tipos diferentes
  • A exceção pode carregar dados quaisquer

//classe estatica

            else
            {
                return duration.TotalDays.ToString("F1", CultureInfo.InvariantCulture) + "days";
            }
        }
    }
